7.3 Multi-Select Questions (MS)

Multi-Select questions present a list of items to the student, from which they should choose one or more correct ones (or a pre-defined number of choices, if the score field is set to "limited").

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "MS", "Multi-Select" or "Multi Select Question" are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
shuffle Whether the possible answer choices are shown in a shuffled order or not ("yes" or "no"). Defaults to yes.
score x The scoring approach to use; must be specified before adding possible selections. A single field is expected, with one of the following possible specifications:
  • "equal": the total points for the question are equally distributed among each of the selections;
  • "all": all-or-nothing, i.e. the student must choose all the correct selections to receive the full question marks;
  • "rmw": right-minus-wrong: same as "equal", but with negative scoring; zero is the minimum score (e.g. if there were four matching options, and one was correct and three incorrect, the total score would be 0 points).
  • "limited": the total points for the question are distributed across correct answers only, and the number of selections allowed is limited to the number of correct answers.
selection x A possible selection. At least one must be specified. Contains two fields: a digit (1 or 0), signifying whether this is a correct selection or not, and its text.
enumeration The type of enumeration shown next to each option:
1 = 1,2,3,4,5; 2 = i,ii,iii,iv,v; 3 = I,II,III,IV,V; 4 = a,b,c,d,e; 5 = A,B,C,D,E; 6 = none.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
