7.6 Multi Short Answer Questions (MSA)

MSA questions ask a question, and then request several possible answers to that question. They are different from SA questions with multiple blanks:

  • SA questions with multiple blanks have one or more specific responses defined for each blank;
  • MSA questions define a set of possible responses, which can be placed in any blank.

In short, SA questions with multiple blanks are like FIB Questions, whereas MSA questions are of the type "give n examples of ...". Also note that, unlike SA or FIB questions, MSA have native support for blanks with several rows (this may change in the future).

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "MSA", "Multi Short Answer", or other variants are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
image Insert an image at the start of the question. Specifies a single field, which is the image location.
blanks The number of blanks: 1 (default) or more. Can be followed by the number of columns (20, 40 or 60), and the number of rows (1 to 6) used for all blanks.
response x A correct (or partially correct) answer. Must define at least as many as there are blanks. It specifies the score first (percentage of total question score this response is worth), followed by the expected response, and then the type of answer specified. The type can be:
  • "case": the response is case-sensitive;
  • "nocase": the response is not case-sensitive;
  • "regex": the response is a regular expression.
There is no limit to the number of responses given per blank. If more than one response matches the given answer, the first one declared is used.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
