7.1 True/False Questions (TF)

True-False questions are like a simplified version of Multiple-Choice questions, where there are only two possible choices, True or False. Typically one of these is incorrect, and the other correct.

The following is a simple example of a CSV file containing the definition of a True/False question:

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "TF", "True-False", "True False Question" and other variations are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Defaults no "no".
ismandatory If this is a mandatory question (yes) or not (no). Defaults to "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
true x Defines the percentage of the question score that the student will get, if he chooses True. Can have an optional second field defining the feedback shown to the student at the end of the test, for this option.
false x Defines the percentage of the question score that the student will get, if he chooses False. Can have an optional second field defining the feedback shown to the student at the end of the test, for this option.
enumeration The type of enumeration shown next to each option:
1 = 1,2,3,4,5; 2 = i,ii,iii,iv,v; 3 = I,II,III,IV,V; 4 = a,b,c,d,e; 5 = A,B,C,D,E; 6 = none.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
