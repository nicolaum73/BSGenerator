7.2 Multiple-Choice Questions (MC)

Multiple-Choice questions are by far the most commonly used type of question in online tests. They are similar to True-False questions, but can have any number of possible choices.

The following is a simple example of a CSV file containing the definition of a Multiple-Choice question:

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. ``MC'', ``Multiple Choice'' or ``Multiple Choice Question'' are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Default to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
shuffle Whether the possible answer choices are shown in a shuffled order or not (``yes'' or ``no''). Defaults to yes.
choice x Possible MCQ choice. You must define at least one (but probably four or more). It defines the score first, followed by the label that appears to the student. Note that the score is a percentage of the question score (defined with weighting, but possibly overridden in quizzes). There is no limit to the number of possible answers, although too many may become hard to visualise. Use one choice definition per possible answer. Can include HTML formatting.
enumeration The type of enumeration shown next to each option:
1 = 1,2,3,4,5; 2 = i,ii,iii,iv,v; 3 = I,II,III,IV,V; 4 = a,b,c,d,e; 5 = A,B,C,D,E; 6 = none.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
