7.9 Ordering Questions (ORD)

Ordering questions present a list of items to the student; for each item, an order (from a drop-down box) must be selected. They are like Matching questions, except that the possible answers are 1, 2, ..., n, where is the number of items presented.

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "ORD", "Ordering" or "Ordering Question" are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
image Insert an image at the start of the question. Specifies a single field, which is the image location.
item x Defines one of the items to sort; at least one must be defined. It requires two fields: the correct order for this item (1-n, where n is the number of items defined), and the text of the item.
score x The scoring approach to use. A single field is expected, with one of the following specifications:
  • "equal": the total points for the question are equally distributed among each of the items to order;
  • "all": all-or-nothing, i.e. the student must order all the items correctly to receive the overall points assigned to this question;
  • right-minus-wrong: same as "equal", but with negative scoring; zero is the minimum score (e.g.if there were four ordering items, and one was correct and three incorrect, the total score would be 0 points).
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
