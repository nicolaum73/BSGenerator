7.7 Fill-in-the-Blanks Questions (FIB)

Fill-in-the-Blanks questions essentially represent the same kind of question as SA questions with multiple blanks. However, while the latter call each of the blanks "Blank # 1","Blank # 2", etc, FIB questions allow you to insert the blanks within the text of the question. They also allow partial correctness for each response (although they do not have the choice of scoring alternatives as SA questions with multiple blanks do).

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "FIB", "Fill in the Blanks", or "Fill-in-the-Blanks" are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting. Contrary to all other question types, there can be several text definitions, interspersed with blank definitions.
image Insert an image at the start of the question. Specifies a single field, which is the image location.
blank x Defines a blank for the student to enter a response; at least one must be defined. Can be placed before, in the middle, or after other text and blank definitions. Blanks are automatically labelled 1, 2, etc. Can be specified with a number of columns (e.g. blank,10) or without (blank,), in which case its dimensions default to 30x1. Note that Brightspace only directly suports blanks with 5-30 columns (in multiples of 5), and a single line.
response x A correct (or partially correct) answer, assigned to a blank. If there are blanks without responses assigned to them, the question must be corrected manually.
It specifies the score first (percentage of question points this response is worth), followed by the response, the type of response specified, and an optional label of the blank it refers to (defaults to 1, i.e. the first blank).
The type can be:
  • "case": the response is case-sensitive;
  • "nocase": the response is not case-sensitive;
  • "regex": the response is a regular expression.
There is no limit to the number of responses given per blank. If more than one response matches the given answer, the first one declared is used.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
