7.5 Short Answer Questions (SA)

Short Answer questions ask for one or more short answers (textual or numeric). These are then compared to pre-defined answers, for each blank.

In Brightspace, single blank short answer and multiple blank short answer questions are somewhat different, even though they both are defined under the same question type. Single blank present a single text-entry box for the user to type their answer; the score for that answer reflects the total score for the question. For multiple blank questions, there are different scoring options.

Note that BSGenerator provides some extensions to the online-editor used by Brightspace, for single blank questions, in that not-fully-correct answers can be specified.

Multiple blank short answers defined through SA questions seem somewhat of an after-thought in Brightspace, and are much better implemented either through Multi Short Answer or Fill-in-the-Blanks questions.

The fields that can be specified are as follows:

FieldMandatoryDescription
questiontype x Must be the first specified field. It states the type of the question. "SA", "Short Answer", or "Short Answer Question" are all accepted.
title The title of the question. Defaults to "questiontitle".
weighting The score of the question, between 0 and 9999. Defaults to 1.
difficulty The difficulty of the question (1-5). Defaults to 1.
isbonus If this is a bonus question (yes) or not (no). Default no "no".
ismandatory If this is a mandatory question (yes) or not (no). Default no "yes".
text x The text of the question. Must be contained within double quotes, if extending more than one line, or if containing commas. Can include HTML formatting.
score The scoring to use; defaults to "equal". Must be set before specifying blanks or responses. Can be set to:
  • "equal": the question points are equally distributed among all blanks (use this for single blank questions);
  • "all": all-or-nothing, i.e. the student must correctly answer all the blanks to receive the full question marks.
blanks The number of blanks: 1 (default, for single blank) or more.
response x A correct (or partially correct) answer. Must define at least one. It specifies the score first, followed by the expected response, and then the type of answer specified; an optional fourth field defines the blank this response refers to (defaults to 1, i.e. the first blank).
For single blank questions, the score is a percentage of the total question points. For multiple blank questions, if the score is set to "equal", then the response score must be set to 1/n for n blanks (e.g. 0.5 for two blanks); if the score is set to "all", then the response scores must all be set to "0". The type can be:
  • "case": the response is case-sensitive;
  • "nocase": the response is not case-sensitive;
  • "regex": the response is a regular expression.
There is no limit to the number of responses given per blank. If more than one response matches the given answer, the first one declared is used.
hint A hint shown to the students when answering the question in a quiz. Can include HTML formatting.
feedback Feedback to show to the students, when visualising their test and answers after taking the test (it is not shown during test taking). Can include HTML formatting.
keyword Defines a keyword. Can have an optional second field with a value. These are not uploaded to Brightspace.
keywords Same as before, but several keywords can be defined. No value will be associated with each keyword.
