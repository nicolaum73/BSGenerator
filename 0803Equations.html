8.3 Creating Equations

Brightspace can easily convert latex equations into MathML for display, and BSGenerator provides the same funcionality. The following CSV file illustrates several ways in which equations can be incorporated into a question:

To create inline equations (i.e. equations that typically have text around them, but also simple equations), place the corresponding latex equation code between "\\(" and "\\)" tokens. If creating full equations (i.e. equations that are typically centered on their own in a line of text, usually complex equations), place the latex equation code between "\\[" and \\]" tokens, or between "$$" tokens.
Note any backslash must be "escaped" with another backslash, hence using "\\[" and "\\]" as tokens. This also applies to equation code, such as "\\sqrt". Finally, if your latex equation code is inside a Python string, use raw strings to ensure that backslashes are not interpreted as Python string escaping characters. The following CSV definition gives an example:

latex2mathml

BSGenerator versions prior to v1.1 suggested the use of this Python library for generation of equations. While it technically will still work, its use is discouraged from v1.1 onwards.
